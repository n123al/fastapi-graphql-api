# Security Policy

## Supported Versions

We aim to support the latest release of the project. Security fixes are prioritized for the current main branch.

## Reporting a Vulnerability

- Please report security issues privately.
- Use GitHubâ€™s Private Vulnerability Reporting if available, or email the maintainers.
- Provide a clear description, steps to reproduce, affected versions, and any relevant context.

## Response Process

- We will acknowledge receipt within 72 hours.
- We will investigate and determine impact and scope.
- We will work on a fix and coordinate a disclosure timeline.
- We will publish a security advisory and a patched release.

## Best Practices

- Do not publicly disclose vulnerabilities before a fix is released.
- Avoid sharing sensitive information in public issues.
- Use supported versions and keep dependencies updated.

Thank you for helping keep the community secure.
